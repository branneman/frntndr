The Grunt buildscript explained

First, make sure you have a basic understanding of what Grunt does, go read the Grunt Getting Started guide.

The default task

The build task, does several things in the following order: Sass compilation, autoprefixer, copying all the files into the build/ directory, minifying images, removing the Swig includes (replacing them with the actual html) and minifying javascript with UglifyJS.

The watcher task

This internal task is automatically active when the node server is started, and remains active. Output or errors from Sass or Autoprefixer are printed to the command line.

This task is not intended to be started manually!

The test task

This task runs both JSHint and Jasmine, in that order. You can find the configuration for JSHint in the file .jshintrc The Jasmine specs are located in /src/static/js/spec/

The zip task

This task zip's the entire build/ directory. The default build has to be ran first.

The deploy task

This task uploads the entire content of the build/ directory to the ftp server. Hence, it can't upload anything if you didn't run the build first.

For it to work, you need to enter your credentials in the .ftppass file, and your ftp server info in the config.json file.
